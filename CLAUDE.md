## 3. UI/UX 디자인 가이드

### 레퍼런스

- **profgalloway.com** — 매거진형 아티클 피드, 사이드바에 프로필, 하단 구독 CTA
- **Kellogg Insight** — 학술 연구를 실무자 언어로 변환하는 아티클 구조

### 디자인 원칙

1. **콘텐츠 퍼스트:** 아티클이 곧 사이트. 불필요한 장식 최소화
2. **데이터 시각화가 시그니처:** Galloway의 일러스트 = BDM Lab의 그래프/차트
3. **읽기 경험 최적화:** 본문 최대폭 제한 (680–720px), 충분한 행간 (1.8–2.0), serif 계열 본문 폰트
4. **라이트 모드 기본:** 밝고 깨끗한 배경. 포인트 컬러 최소 사용
5. **모바일 퍼스트:** 인스타 → 웹 유입이 주요 경로이므로 모바일 경험 우선

### 컬러 시스템


### 핵심 컴포넌트

- **하단 CTA:** 뉴스레터 구독 유도 (Galloway 스타일)

#### 아티클 페이지

## 4. 기술 스택

| 요소 | 선택 | 이유 |
|------|------|------|
| 프레임워크 | **Next.js (App Router)** | SSG로 빠르고 SEO/AEO에 강함 |
| 스타일링 | **Tailwind CSS** | 바이브 코딩 친화적, 유틸리티 기반 |
| 콘텐츠 | **MDX 파일** | Markdown + 컴포넌트(그래프 등) 삽입 가능 |
| 배포 | **Vercel** | Next.js 네이티브 배포, 무료 티어 |
| 뉴스레터 | **Buttondown** 또는 **Mailchimp** | 무료 → 유료 전환 가능 |
| 분석 | **Vercel Analytics** 또는 **Plausible** | 프라이버시 친화적 |

### 콘텐츠 관리 방식

```
/content
├── /articles
│   ├── 2026-03-02-instagram-algorithm.mdx
│   ├── 2026-02-23-location-data.mdx
│   └── ...
└── /authors
    └── professor.json
```

각 MDX 파일의 frontmatter:

```yaml
---
title: "인스타가 당신의 팔로워를 보여주지 않는 진짜 이유"
series: "algorithm-decode"        # 시리즈 슬러그
tags: ["algorithm", "platform", "instagram"]
date: "2026-03-02"
readingTime: 8                    # 분
excerpt: "팔로워 수가 도달률과 무관해진 건 버그가 아니라 설계다."
featured: true                    # 홈 히어로에 표시
---
```

## 5. SEO / AEO 최적화

### 메타 구조

- 모든 페이지에 `<title>`, `<meta description>`, OG 태그, JSON-LD 구조화 데이터
- 아티클: `Article` 스키마 (author, datePublished, publisher)
- About: `Person` + `Organization` 스키마
- 사이트 전체: `WebSite` 스키마 + `sameAs` (소셜 링크)

### AEO 전략

- 각 아티클에 명확한 질문-답변 구조 포함 (LLM이 참조하기 쉬운 형태)
- "빅데이터 마케팅 랩"이 AEO에서 고유 엔티티로 인식되도록 일관된 브랜드명 사용
- 시리즈명("알고리즘 디코드", "랩 리서치")도 반복 노출로 엔티티화

## 6. 콘텐츠 톤 가이드

### 아티클 작성 원칙

1. **데이터 기반:** 주장에는 반드시 수치/출처가 동반
2. **과장 금지:** "혁명적", "놀라운" 같은 형용사 자제. 사실로 말함
3. **실무 번역:** 학술 용어를 실무자가 이해할 수 있는 언어로 변환
4. **구조:** 문제 → 메커니즘(왜 그런가) → 근거(데이터) → 시사점(그래서 어쩌라고)
5. **한계 인정:** 분석의 한계를 솔직히 밝힘 (이게 차별점)

### 아티클 포맷

```
[시리즈 라벨]
[제목 — 호기심 유발하되 과장 없이]
[메타: 저자 · 날짜 · 읽기시간]

[리드 — 핵심 발견 2-3문장]

[섹션 1: 현상/문제]
[섹션 2: 메커니즘/원인 — 데이터 시각화 포함]
[섹션 3: 시사점/액션]
[섹션 4: 한계/추가 질문]

[출처/참고문헌]
```
