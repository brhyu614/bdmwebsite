# 06 · Studio 아키텍처 — 집단지성 버전 팩토리

**결정일: 2026-04-21**

Content Factory의 본질은 "자동 완결"이 아니라 **"집단지성 버전 팩토리"**. AI가 끝내는 게 아니라, 사람이 피드백을 큐레이팅하며 여러 번 버전을 돌려 최종안을 만드는 구조.

## 4개 핵심 조각

### 1. 브랜드 프롬프트 (system-level, 고정)

교수님 voice / 어투 / 규칙. 모든 생성에 자동 주입. GPT Playground의 system prompt처럼 영구 고정.

- **저장 위치:** `brand_voices` 테이블 (신규) or `styles.system_prompt` 필드
- **편집 주체:** 교수님 본인만
- **적용 범위:** 모든 스타일·모든 생성에 선행 적용

### 2. 스타일 라이브러리 (user-level, 편집 가능)

따라하고 싶은 캐러셀 or 본인 과거 작업물을 레퍼런스로 저장. 여러 개 두고 선택/믹스해서 재생산.

- **저장 위치:** `styles` 테이블 + `style_items` 테이블 (스크린샷·분석 JSON)
- **편집:** 추가/편집/삭제 UI 필요 (기존 "편집 금지" 방침 2026-04-21 번복)
- **선택:** 생성 시 하나 선택 or 복수 믹스

### 3. 피드백 큐레이션

Gemini 자동 평가는 "제안 모음"일 뿐. 유저가 각 제안에 ✅/❌ 체크하고 + 자유 입력으로 추가 지시. 선택된 것만 regenerate 프롬프트에 반영.

- **UI:** 체크박스 per 제안 + 자유 입력란
- **저장:** `improvements` 테이블 — feedback_text (선택된 제안 + 자유 입력 합침)
- **주의:** 자동 반영 금지. "자동으로 다 해주는" 방향은 교수님이 원하지 않음.

### 4. 버전 히스토리

같은 topic에 v1, v2, v3… 누적. 각 버전에 "적용된 피드백 / 거부된 피드백" 기록. "최종 컨펌" 플래그로 버전 고정. 누적 피드백 패턴이 장기적으로 brand_voice 업데이트에 반영됨.

- **저장:** `generations` 테이블의 version 컬럼 + `improvements`의 from/to_version
- **UI:** 버전 목록 + diff 뷰 + "최종 컨펌" 버튼 + "스타일에 반영하기" 버튼
- **데이터 가치:** 삭제 아닌 archive (과거 피드백이 learning data)

### 4-1. "스타일에 반영하기" 버튼 (2026-04-21 추가)

**집단지성 학습 루프의 핵심.** 최종 컨펌한 생성물은 유저가 명시적으로 "스타일에 반영하기" 버튼을 누르면 해당 스타일의 새 `style_item`으로 추가됨 → 다음 생성부터 이 스타일이 업데이트된 레퍼런스를 반영.

- **자동 반영 금지.** 유저가 명시적으로 클릭해야 반영.
- **언제:** 최종 컨펌 이후 아무 때나 (작업 중에도 별도로 스타일 관리 가능)
- **동작:** 현재 선택된 스타일의 style_items에 generation.output을 synthetic item으로 append → styles.analyzed 재분석 큐잉
- **효과:** 유저 본인이 승인한 버전들이 점점 쌓이면서 스타일이 본인 작풍에 수렴

## 현재 상태 (2026-04-21)

- ✓ `styles` 테이블·`style_id` 파라미터·`regenerate` 모드 (뼈대 존재)
- ✗ brand_voice 분리 저장
- ✗ 스타일 라이브러리 편집 UI (현재는 레퍼런스 0개 기본 스타일만)
- ✗ 피드백 큐레이션 UI (Gemini 제안이 읽기 전용으로만 표시)
- ✗ 버전 히스토리 뷰 / 최종 컨펌 플래그

## 빌드 순서 (합의됨 2026-04-21)

1. **스타일 라이브러리 UI** — 레퍼런스 추가/편집/선택. 가장 중요. 이거 없으면 다 "기본 스타일"로만 생성됨
2. **피드백 큐레이션 + 재생성** — 체크박스 + 자유 입력 → regenerate 호출 (버전 번호 자동 증가)
3. **버전 히스토리 뷰 + "최종 컨펌"** — 같은 topic 내 v1/v2/v3 비교
4. **이미지 렌더링 레이어** — Pexels/Unsplash 연동 + 슬라이드 템플릿 + 1080x1350 최종 미리보기 (폰트·크기·배경 체크)

## 설계 원칙 (신규 기능 추가 시)

- 신규 기능은 4조각 중 어디 속하는지 먼저 분류
- "자동으로 다 해주는" 방향 지양 → 항상 유저 개입 지점 확보
- 스타일/브랜드 프롬프트는 분리 저장 (한쪽 수정이 다른 쪽 영향 주지 않게)
- 버전은 삭제 아닌 archive
