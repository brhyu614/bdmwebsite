# BDM Content Factory — 사용 매뉴얼

> BDM Lab 소셜미디어 콘텐츠 자동화 프로그램
> 웹사이트 글 하나를 캐러셀, 네이버 블로그, 릴스, Threads로 자동 변환

---

## 1. 프로그램이 뭔가요?

교수님이 쓴 글(웹사이트 아티클, 책 원고 등)을 넣으면, AI가 자동으로 4개 플랫폼용 콘텐츠로 변환해주는 프로그램입니다.

**입력:** 글 + 이미지
**출력:** 인스타 캐러셀 / 네이버 블로그 / 릴스 스크립트 / Threads 포스트

---

## 2. 시작하기

### 파일 위치
```
BDM-Assets/content-factory.html
```

### 여는 법
파일을 더블클릭하면 브라우저에서 열립니다. (Chrome 권장)

### 첫 실행: API 키 입력
1. 왼쪽 사이드바 상단에 **API KEY** 입력란이 있습니다
2. OpenAI API 키를 입력합니다 (sk-proj-... 로 시작하는 키)
3. 오른쪽 초록 점이 켜지면 준비 완료
4. **한 번만 입력하면 됩니다** — 브라우저가 기억합니다

> API 키는 교수님에게 받으세요. 코드에 적거나 다른 사람에게 공유하지 마세요.

**사용법:**

**방법 A — 파일에서 복사 (권장):**
1. `content/articles/` 폴더에서 오늘 할 글의 `.mdx` 파일을 텍스트 에디터로 열기
2. `---` 아래의 본문 전체를 복사
3. Content Factory에서 "직접 입력" 탭 클릭
4. **제목**: 위 표에서 제목 복사
5. **본문**: 복사한 본문 붙여넣기

**방법 B — 웹사이트에서 복사:**
1. bigdatamarketinglab.com 접속
2. 해당 글 페이지에서 본문 전체 드래그 → 복사
3. Content Factory "직접 입력" 탭에 붙여넣기

**방법 C — 교수님이 준 텍스트:**
1. 교수님이 카톡/이메일로 보내준 글을 그대로 붙여넣기
2. 짧은 주제 한 줄도 가능 (예: "에이전틱 커머스에 대해서")

### Step 2: 이미지 넣기

1. **IMAGES** 영역에 이미지를 드래그앤드롭
2. 또는 클릭하여 파일 선택

**이미지 규칙:**
- 가능한 많이 넣으세요 (10-15장 권장). AI가 골라서 배치합니다
- **커버용 이미지**는 파일명을 `cover_` 로 시작하세요
  - 예: `cover_01.jpg`, `cover_쇼핑화면.png`
  - 커버 이미지는 초록 테두리 + COVER 뱃지로 표시됩니다
- 나머지 이미지는 파일명 아무거나 상관없습니다
- 이미지 없이도 실행 가능합니다 (AI가 Pexels 키워드로 대체)

### Step 3: 플랫폼 선택

4개 플랫폼 중 원하는 것을 선택합니다. (기본: 전체 선택)
- 📸 **캐러셀** — 인스타그램 캐러셀 (6-8장 슬라이드)
- 📗 **네이버** — 네이버 블로그 포스트 (SEO 최적화)
- 🎬 **릴스** — 인스타그램 릴스 스크립트 (45-60초)
- 🔗 **Threads** — Threads 짧은 포스트

클릭하면 선택/해제됩니다. 초록 테두리 = 선택됨.

### Step 4: 생성

**"생성하기"** 버튼을 클릭합니다.

- 플랫폼당 10-20초 소요
- 진행 상황이 하단에 표시됩니다
- 전체 4개 하면 약 1분 소요

### Step 5: 결과 확인

오른쪽 미리보기 영역에 결과가 표시됩니다.

**탭으로 플랫폼별 확인:**
- 📸 캐러셀 — 슬라이드별로 보여줌 (커버 → 본문 슬라이드 → 아웃트로)
- 📗 네이버 — 제목, 섹션별 본문, 해시태그
- 🎬 릴스 — 씬별 스크립트 (시간, 태그, 화면 텍스트, 나레이션)
- 🔗 Threads — 포스트 전체 텍스트

## 4. 피드백 & 재생성 (교수님과 함께)

### 피드백 넣는 법

1. 결과물을 교수님에게 보여줍니다
2. 교수님의 피드백을 받습니다
3. 하단 **"피드백"** 영역의 텍스트 입력란에 피드백을 그대로 입력합니다

**피드백 예시:**
```
캐러셀 훅이 약해. 숫자로 시작해야 함. "87%의 마케터가 모르는 사실" 같은 느낌으로.
네이버 서두가 딱딱해. 대화체로 바꿔.
릴스 OK.
Threads 너무 길어. 핵심 한 문장만.
```

4. **"피드백 반영하여 재생성"** 버튼 클릭
5. AI가 피드백을 반영한 새 버전을 만듭니다
6. 오른쪽 상단에 버전 표시: v1 → v2 → v3...

### 승인할 때

교수님이 "OK" 하시면:
1. **"최종 승인 ✓"** 버튼 클릭
2. 피드백에서 학습 포인트가 자동 추출됩니다
3. 다음 생성 때 AI가 이 규칙을 기억합니다

**중요:** 피드백이 있어야 시스템이 똑똑해집니다. 교수님이 아무 피드백 없이 OK하면 학습 포인트가 없습니다.

## 5. 결과물 내보내기

각 플랫폼 결과 아래에 복사 버튼이 있습니다:

| 플랫폼 | 복사 버튼 | 어디에 붙여넣나 |
|--------|----------|-------------|
| 캐러셀 | 📋 YAML 복사 | `make_carousel.py` 입력 파일로 저장 |
| 네이버 | 📋 JSON 복사 | `naver-blog-generator.html`에서 import |
| 릴스 | 📋 JSON 복사 | `reels-generator.html`에서 참고 |
| Threads | 📋 텍스트 복사 | Threads 앱에 직접 붙여넣기 |

### 캐러셀 결과물로 실제 캐러셀 만드는 법

1. "📋 YAML 복사" 클릭
2. 텍스트 에디터에 붙여넣고 `input.yaml`로 저장
3. 터미널에서:
```bash
cd ~/Dropbox/Website-BigDMKTG/BDM-Assets/carousel-generator
python3 make_carousel.py input.yaml
```
4. `carousel-output/` 폴더에 MP4 슬라이드가 생성됩니다

## 6. 누적 학습 규칙

왼쪽 사이드바 하단에 **"누적 학습 규칙"** 이 표시됩니다.

- 교수님의 피드백에서 자동으로 추출된 규칙입니다
- 예: "[carousel] 훅은 반드시 숫자로 시작", "[naver] 서두는 대화체로"
- 이 규칙은 다음 생성 때 AI에게 자동으로 전달됩니다
- 브라우저를 닫아도 유지됩니다 (localStorage에 저장)

### 학습 규칙 초기화가 필요한 경우
브라우저 개발자 도구(F12) → Console에서:
```javascript
localStorage.removeItem('bdm_factory_learnings');
```

## 7. 매일 루틴 (RA 체크리스트)

### 하루 하나 — 네이버 블로그 + 캐러셀

아티클 목록(위 표)에서 1번부터 순서대로 하루에 하나씩 진행합니다.

```
□ 1. 오늘 할 글 확인 (목록에서 순서대로)
□ 2. content/articles/ 폴더에서 해당 .mdx 파일 열기
□ 3. 글 주제에 맞는 이미지 10-15장 검색/준비
       - Unsplash, Pexels, Google 이미지 등에서
       - 커버용 2-3장은 파일명 cover_ 로 시작
       - 나머지는 본문 삽입용
□ 4. Content Factory 열기 (content-factory.html)
□ 5. "직접 입력" 탭 → 제목 + 본문 붙여넣기
□ 6. 이미지 드래그앤드롭
□ 7. 플랫폼 선택 (네이버 + 캐러셀 최소)
□ 8. "생성하기" 클릭 → 1분 대기
□ 9. 결과물을 교수님에게 전달 (카톡 or 슬랙)
□ 10. 교수님 피드백 받기
□ 11. 피드백 입력 → "피드백 반영하여 재생성" 클릭
□ 12. OK 받으면 → "최종 승인" 클릭
□ 13. 결과물 내보내기:
       - 네이버: JSON 복사 → naver-blog-generator.html에서 마무리 → 포스팅
       - 캐러셀: YAML 복사 → make_carousel.py 실행 → MP4 생성 → 인스타 업로드
       - Threads: 텍스트 복사 → Threads 앱에 바로 포스팅
```

**소요 시간 예상:**
- 처음 몇 번: 글당 1-2시간 (피드백 주고받기 포함)
- 10개 정도 하고 나면: 글당 30분-1시간 (학습 규칙이 쌓여서 피드백 줄어듦)
- 20개 이후: 글당 20-30분 (거의 한 번에 OK)

### 3-4주 플랜

| 주차 | 할 글 | 비고 |
|------|-------|------|
| 1주차 | #1-#5 | 처음이라 피드백 많을 수 있음. 학습 기간 |
| 2주차 | #6-#11 | 학습 규칙 5-10개 쌓임. 품질 개선 느껴짐 |
| 3주차 | #12-#17 | 거의 한 번에 OK 나오기 시작 |
| 4주차 | #18-#23 | 마무리. 새 글이 올라오면 바로 변환 가능 |

## 8. 자주 묻는 질문

### Q: 생성이 안 됩니다
- API 키가 올바른지 확인 (sk-proj-... 형태)
- 인터넷 연결 확인
- 글 본문이 비어있지 않은지 확인

### Q: 결과물 품질이 안 좋습니다
- 본문을 더 많이 넣으세요 (짧은 주제보다 전체 글이 훨씬 나음)
- 피드백을 구체적으로 입력하세요 ("별로" 보다 "숫자로 시작해, 질문형으로" 같은 구체적 피드백)
- 피드백이 쌓일수록 자동으로 개선됩니다

### Q: 이미지를 안 넣으면?
- 괜찮습니다. AI가 Pexels 검색 키워드를 자동 생성합니다
- 하지만 직접 고른 이미지가 훨씬 좋은 결과를 만듭니다

### Q: API 키를 매번 입력해야 하나요?
- 아니요. 한 번 입력하면 브라우저가 기억합니다
- 다른 브라우저나 컴퓨터에서는 다시 입력해야 합니다

### Q: 이전 결과를 다시 볼 수 있나요?
- 현재 버전은 브라우저를 닫으면 결과가 사라집니다
- 중요한 결과는 반드시 복사(📋 버튼)해서 저장하세요

## 9. 파일 구조

```

├── content-factory.html        ← 이 프로그램 (브라우저에서 열기)
├── content-factory/
│   ├── factory.py              ← 터미널 버전 (고급 사용자용)

│   ├── output/                 ← 터미널 버전 결과물
```
