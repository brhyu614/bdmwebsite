# BDM Content Factory — 벤치마킹 & 장점 활용 요약

> 기존 오픈소스, SaaS, GitHub Skill 등을 조사하여 각각의 장점을 우리 프로그램에 어떻게 반영했는지 정리

---

## 1. GitHub 오픈소스 프로젝트

### claude-repurpose (GitHub, 16★)
- **뭐하는 도구:** 글 하나를 17개 플랫폼용 텍스트로 변환하는 Claude Code 스킬
- **장점:** 플랫폼마다 톤/길이/형식을 다르게 지정하는 프롬프트 구조가 체계적
- **단점:** 텍스트만 나옴. 비주얼 없음. 한국어 미지원. 피드백 시스템 없음
- **우리가 가져온 것:**
  - "하나의 글 → 여러 플랫폼" 프롬프트 구조
  - 플랫폼별로 톤/길이/형식을 다르게 지시하는 시스템 프롬프트 설계 방식
  - 7일 퍼블리싱 캘린더 개념 → 향후 스케줄 기능으로 확장 가능

### content-pipeline (GitHub, 102★)
- **뭐하는 도구:** 중국 블로거가 위챗/샤오홍슈 운영 자동화용으로 만든 Claude Code 스킬. 2개월 실운영
- **장점:** HTML 템플릿 → 브라우저 렌더링으로 실제 캐러셀 이미지 생성. 슬래시 커맨드 하나로 실행
- **단점:** 중국 플랫폼 전용. 정적 이미지만. 피드백 루프 없음. 1인용
- **우리가 가져온 것:**
  - 슬래시 커맨드 하나로 실행하는 단순 UX → 우리는 "버튼 하나"로 생성
  - HTML 렌더링으로 시각적 미리보기 → 프로그램 안에서 슬라이드별 미리보기
  - 커버 이미지 + 본문 삽화 자동 생성 개념 → Pexels 연동 이미지 삽입

### ai-marketing-skills (GitHub, 1,687★)
- **뭐하는 도구:** Single Grain CEO가 만든 마케팅 스킬 모음. Content Ops, Podcast Ops, SEO Ops 포함
- **장점:** "Expert Panel" 자동 채점 + "Quality Gate" (점수 미달 시 재생성). 실제 수백만 달러 매출 파이프라인에서 사용
- **단점:** 영어 B2B SaaS 전용. 콘텐츠 변환이 아닌 전략 중심. 비주얼 생성 없음
- **우리가 가져온 것:**
  - **Expert Panel 자동 채점 시스템** → Gemini가 훅 강도, 데이터 근거, 흐름, CTA 등 항목별 자동 채점
  - **Quality Gate** → 점수 15점 미만이면 "재생성 권장" 경고
  - Content Ops의 채점 루브릭 구조 → 캐러셀/네이버/Threads 각각 맞춤 채점 기준

### yozm.wishket 글 (Claude Code 에이전트 팀)
- **뭐하는 도구:** Mac Mini에 5개 AI 에이전트(CMO, 작가, SNS 등)를 배치하여 매시간 자동 마케팅 운영
- **장점:** "채용 공고를 쓴다고 생각하라" 철학. rules/ 폴더로 공통 규칙 관리. 파일 기반 소통. 에이전트 메모리
- **단점:** CMS 발행 중심. 콘텐츠 변환 아님. 영어 전용. 팀 협업 구조 없음
- **우리가 가져온 것:**
  - **에이전트를 "채용"하듯 설계** → 각 프로그램의 시스템 프롬프트를 직책+업무범위+도구+규칙으로 구성
  - **rules 파일 분리** → prompts/ 폴더에 프롬프트를 txt로 분리하여 누구나 수정 가능
  - **파일 기반 소통** → JSON/YAML/txt로 프로그램 간 데이터 교환
  - **자동 학습** → 피드백에서 규칙 추출 → 다음 생성에 반영 (에이전트 메모리 개념)

### seo-geo-claude-skills (GitHub, 900★)
- **뭐하는 도구:** 20개 SEO/GEO 스킬 (키워드 리서치, 콘텐츠 작성, AEO)
- **장점:** CORE-EEAT + CITE 프레임워크. AEO 최적화 전문
- **단점:** 영어 전용
- **우리가 가져온 것:**
  - **AEO 최적화 규칙** → 네이버 프롬프트에 "AI가 인용하기 좋은 질문-답변 구조" 규칙 내장
  - 핵심 문장 단독 단락 분리, 데이터 문장 앞배치 등 AEO 기법

### avoid-ai-writing (GitHub, 810★)
- **뭐하는 도구:** AI 글쓰기 패턴을 감지하고 제거
- **우리가 가져온 것:**
  - BDM Lab 톤 가이드에 "AI스러운 표현 금지" 규칙 반영
  - "혁명적", "놀라운" 같은 AI 상투어 명시적 금지

## 2. 상용 SaaS 도구

### Distribution.ai (월 $45~)
- **뭐하는 도구:** 블로그 URL → 10개 소셜 포스트 자동 생성 + 3개월 자동 스케줄링
- **장점:** RSS 자동 감지. 브랜드 보이스 학습. A/B 테스트 내장
- **단점:** 한국어 미지원. 네이버 없음. 텍스트만
- **우리가 가져온 것:**
  - **브랜드 보이스 내장** → BDM Lab 톤가이드를 시스템 프롬프트에 상세 반영
  - **새 글 자동 감지 개념** → 아티클 목록에서 선택하는 UX (향후 자동화 가능)
  - 피드백 누적 = 브랜드 보이스 학습과 동일한 효과

### PostNitro / Contentdrips (캐러셀 전문)
- **뭐하는 도구:** 주제/URL → 캐러셀 이미지 자동 생성
- **장점:** API 제공 (자동화 가능). 브랜드 키트 적용
- **단점:** 정적 이미지만. 한국어 약함. 월 $15-26
- **우리가 가져온 것:**
  - "URL/텍스트 하나만 넣으면 바로 캐러셀" UX 단순함
  - 브랜드 키트(색상, 폰트, 로고) 일관 적용 → accent #39FF14, BDM Lab 브랜딩 통일

### Lately AI (SaaS)
- **뭐하는 도구:** 뉴로사이언스 기반 브랜드 보이스 학습 → 콘텐츠 자동 생성
- **장점:** 기존 콘텐츠에서 브랜드 보이스를 학습. 계속 적응
- **단점:** 영어 전용. 엔터프라이즈 가격
- **우리가 가져온 것:**
  - **피드백 누적 학습 시스템** = Lately의 보이스 학습과 같은 개념
  - 교수님 피드백 → 규칙 추출 → 프롬프트에 반영 → 다음 생성에 적용
  - 10-20회 피드백 후 교수님 스타일을 자동 파악

### OpusClip / Munch (영상 자동화)
- **뭐하는 도구:** 긴 영상에서 바이럴 구간 자동 추출 → 숏폼 클립
- **장점:** AI Virality Score. 플랫폼별 트렌드 분석. 80% 사용 가능 클립
- **우리가 가져온 것:**
  - **자동 품질 채점** 개념 → Gemini 자체 점검 시스템 (각 항목 1-5점)
  - 점수 기반 재생성 권장 → Quality Gate

### Gazet AI / BlogWorks (한국 네이버 도구들)
- **뭐하는 도구:** 키워드 → 네이버 블로그 자동 생성 + SEO 최적화 + 자동 포스팅
- **장점:** 네이버 C-Rank/DIA 알고리즘 최적화 전문. 한국어 키워드 분석
- **단점:** 새 글 생성만 (기존 글 변환 불가). SEO 양산형 품질. 학술 톤 부적합
- **우리가 가져온 것:**
  - **네이버 C-Rank/DIA 알고리즘 규칙** → 제목 15-30자, 해시태그 8-12개, 본문 2000자 이상, 키워드 밀도 등
  - SEO 점수 자동 계산 (제목 길이, 본문 길이, 섹션 수, 해시태그 수)

## 3. 자동화 플랫폼

### n8n (오픈소스 워크플로우)
- **뭐하는 도구:** 비주얼 워크플로우 빌더. 524개 소셜미디어 자동화 템플릿
- **장점:** 완전 커스터마이징. 셀프 호스팅. 무료. RSS → AI → 멀티플랫폼 파이프라인
- **우리가 가져온 것:**
  - 파이프라인 구조: 입력 → AI 변환 → 자체 점검 → 피드백 → 출력
  - 향후 n8n/Zapier 연동으로 완전 자동화 가능한 구조로 설계

### Zapier / Make.com
- **우리가 가져온 것:**
  - 트리거 → 변환 → 알림 구조의 기본 개념
  - 향후 "새 MDX 파일 → 자동 생성 → Slack 알림" 확장 가능

## 4. 피드백/학습 시스템 관련

### PromptLayer / Langfuse (프롬프트 버전 관리)
- **뭐하는 도구:** Git처럼 프롬프트를 버전 관리하고 성능 추적
- **우리가 가져온 것:**
  - **프롬프트를 txt 파일로 분리** → prompts/ 폴더에서 버전 관리 가능
  - UI에서도 수정 가능 + localStorage 저장
  - "기본값 복원" 기능으로 실험 후 롤백

### Promptfoo (LLM-as-Judge)
- **뭐하는 도구:** AI 출력물을 다른 AI가 채점
- **우리가 가져온 것:**
  - **Gemini를 Judge로 활용** → Claude가 생성, Gemini가 채점 (모델이 다르면 편향 감소)
  - 항목별 루브릭 채점 (단순 "좋다/나쁘다"가 아닌 1-5점 다차원 평가)
  - 채점 결과 + 구체적 개선 제안 자동 생성

### Data Flywheel 패턴 (NVIDIA)
- **개념:** 더 좋은 프롬프트 → 더 좋은 결과 → 피드백 → 더 강한 평가 → 더 좋은 프롬프트 (선순환)
- **우리가 가져온 것:**
  - 전체 시스템의 핵심 아키텍처:
    1. AI가 콘텐츠 생성
    2. AI(Gemini)가 자체 점검
    3. 교수님이 피드백
    4. 피드백에서 규칙 자동 추출
    5. 다음 생성에 규칙 반영
    6. 반복할수록 품질 상승

## 5. 교수님 강의자료 반영

### 썸네일과 타이틀 (5. 썸네일과 타이틀.pptx, 31슬라이드)
- **핵심 내용:** CTR 극대화를 위한 썸네일/타이틀 전략
- **캐러셀 프롬프트에 반영한 규칙:**
  - 0.1~0.3초 안에 스크롤을 멈추게 해야 함
  - 강한 색 대비 + 명확한 텍스트
  - 호기심 자극: "도대체 왜?" 형태의 질문
  - 숫자/통계로 시작 (CTR 극대화)
  - 텍스트는 적고 강력하게 (3-5단어)
  - 인지 부하 최소화: 핵심 요소만 표현
  - 감정 메시지: 놀람, 의문 유발
  - Before & After 구조 활용
  - 저장/공유를 노린 구조: 숫자/리스트 강조
  - A/B 테스트를 위한 변형 생성

## 6. 우리만의 차별점 (어디에도 없는 것)

| 차별점 | 설명 |
|--------|------|
| **비디오 캐러셀 + 릴스 생성기 보유** | 대부분 도구는 텍스트만 변환. 우리는 이미 작동하는 MP4 생성기가 있음 |
| **한국어 + 네이버 네이티브** | 해외 도구에 전무. 한국 도구들은 SEO 양산형 |
| **학술 데이터 시각화 콘텐츠** | 일반 마케팅 도구에 없는 강점. 차트/데이터가 콘텐츠의 핵심 |
| **듀얼 AI (생성 + 점검)** | Claude 생성 + Gemini 점검. 단일 모델보다 품질 높음 |
| **교수님 피드백 학습 루프** | 쓸수록 좋아지는 시스템. 대부분 도구는 일회성 변환 |
| **프롬프트 + 코드 모두 수정 가능** | txt 파일 직접 편집 + Claude Code로 프로그램 수정. SaaS에선 불가능 |
| **플랫폼별 전용 프로그램** | 범용 도구가 아닌, 캐러셀/네이버/Threads 각각 최적화 |
| **강의자료 기반 썸네일 전략** | 학술적으로 검증된 CTR 최적화 규칙이 프롬프트에 내장 |
