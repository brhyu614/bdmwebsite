# 네이버 블로그 프로그램 가이드

> 담당: 채현
> 프로그램: `program-naver/naver-factory.html`

---

## 프로그램 위치

```
BDM-Assets/
├── program-naver/
│   ├── naver-factory.html          ← 이 파일을 더블클릭해서 브라우저로 열기
│   └── prompts/
│       ├── naver-prompt.txt        ← 생성 프롬프트 (직접 수정 가능)
│       └── review-prompt.txt       ← 자체 점검 프롬프트 (직접 수정 가능)
```

## API 키 입력 (처음 한 번만)

1. 왼쪽 사이드바 상단에 두 개 입력란:
   - **Claude API Key**: 교수님에게 받은 키 입력
   - **Gemini API Key**: 교수님에게 받은 키 입력
2. 초록 점이 켜지면 준비 완료
3. 한 번 입력하면 브라우저가 기억함

## 매일 사용법

### 1. 글 선택
- "웹사이트 글" 탭에서 목록 중 선택하거나
- "직접 입력" 탭에서 `content/articles/` 폴더의 .mdx 파일 본문을 복사해서 붙여넣기

### 2. 이미지 준비
- 글 주제에 맞는 이미지 5-10장 준비 (Unsplash, Pexels 등에서)
- 드래그앤드롭으로 업로드
- 생성 후에도 각 섹션 사이에 이미지를 검색하거나 드래그해서 삽입 가능

### 3. 생성
- "생성하기" 버튼 클릭 (약 15-20초 소요)
- AI가 자동으로:
  - 네이버 SEO 최적화된 블로그 포스트 생성
  - Gemini가 자체 점검 (SEO/서두/깊이/AEO 각 1-5점)

### 4. 결과 확인
- 네이버 블로그처럼 생긴 미리보기
- SEO 점수 자동 계산 (제목 길이, 본문 길이, 해시태그 수 등)
- Gemini 점검 결과 확인

### 5. 교수님 피드백
- 결과물을 교수님에게 보여주기 (스크린샷 or 직접 보여주기)
- 교수님 피드백을 "피드백" 입력란에 그대로 입력
- "피드백 반영 재생성" 클릭
- 만족하면 "최종 승인" 클릭

### 6. 내보내기
- **"완성 HTML 복사"** → 네이버 블로그 에디터에 HTML로 붙여넣기
  - 이미 인라인 스타일 적용됨
  - 시리즈 라벨 박스 + 섹션 + CTA 풋터 포함
- **"JSON 복사"** → 데이터 백업용

## 프롬프트 수정하는 법

### UI에서 수정
1. "프롬프트 설정" 버튼 클릭
2. "생성 프롬프트" 또는 "점검 프롬프트" 탭 선택
3. 내용 수정 → 자동 저장
4. "기본값 복원" 버튼으로 초기화 가능

### 파일로 직접 수정
1. `program-naver/prompts/naver-prompt.txt` 파일을 텍스트 에디터로 열기
2. 수정 후 저장
3. 단, UI에서 저장된 프롬프트가 우선 적용됨 → UI에서 "기본값 복원" 후 파일 수정 내용이 반영

### Claude Code로 프로그램 자체 수정
```bash
cd ~/Dropbox/Website-BigDMKTG/BDM-Assets/program-naver
# Claude Code에서 이런 식으로 요청:
# "naver-factory.html에서 해시태그를 15개로 늘려줘"
# "SEO 점수 계산에 키워드 밀도 항목을 추가해줘"
```

## 네이버 블로그에 올리는 법

1. 네이버 블로그 관리 → 글쓰기
2. 에디터 상단에서 "HTML" 모드로 전환
3. "완성 HTML 복사"로 복사한 내용을 붙여넣기
4. "미리보기"에서 확인
5. 이미지가 Pexels URL이므로 네이버에서 정상 표시됨
6. 해시태그 확인 후 발행

## 매일 체크리스트

```
□ content/articles/ 에서 오늘 할 글 열기
□ 제목 + 본문 복사해서 프로그램에 붙여넣기
□ 관련 이미지 5-10장 준비 + 업로드
□ 생성하기 클릭
□ SEO 점수 확인 (70점 이상이면 OK)
□ 교수님에게 결과물 보여주기
□ 피드백 입력 → 재생성 → 승인
□ "완성 HTML 복사" → 네이버 블로그에 올리기
□ 불편한 점 있으면 content-factory-feedback.md에 기록
```
