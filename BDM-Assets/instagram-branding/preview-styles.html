01 — DIRECTION
BDM Lab vs Algorithm Decode — 어떤 이름이 메인 브랜드가 되어야 하는가
OPTION A
BDM Lab
@bdm.lab · 디스플레이: BDM Lab | 빅데이터마케팅랩
  • 알고리즘 디코드 + 랩 리서치 모든 시리즈 포괄
  • 웹사이트(bigdatamarketinglab.com)와 브랜드 일치
  • 추후 시리즈 추가해도 어색하지 않음
  • 학술적 권위감 + 연구실 정체성
OPTION B
Algorithm Decode
@algorithm_decode · 디스플레이: 알고리즘 디코드
  • 이미 만든 핸들 — 변경 불필요
  • 콘텐츠 정체성이 이름에서 바로 드러남
  • 더 캐치한 이름 (호기심 유발)
  • △ 랩 리서치 콘텐츠 올릴 때 약간 어색

05 — SLOGAN / TAGLINE
"감 대신 근거. 과장 대신 데이터." 기본 철학 + 알고리즘 느낌을 살린 변주들
S1
"감 대신 근거. 과장 대신 데이터."
원본 — 브랜드 철학. 깔끔하고 강함.
S2
"감 대신 근거. 알고리즘 대신 구조."
알고리즘 단어 직접 삽입. 해독 느낌.
S3
"알고리즘을 해부하고, 구조로 증명한다."
알고리즘 + 데이터 분석 정체성.
S4
"감 대신 근거. 과장 대신 구조."
'데이터'를 '구조'로 교체. 알고리즘 해독 톤.
S5
"트렌드를 보지 말고, 구조를 읽어라."
명령형. 도발적. 차별화 강함.
S6
"알고리즘은 감정을 보상한다."
인사이트 자체가 슬로건. 기억에 남음.
S7
"왜 퍼졌는지를 해부합니다."
콘텐츠 정체성 직접 표현.
S8
"구조를 알면, 알고리즘이 보인다."
알고리즘 + 구조 결합. 교육적 톤.
S9
"감 대신 근거. 표면 대신 알고리즘."
'과장'→'표면', '데이터'→'알고리즘' 교체.
S10
"데이터로 해부하고, 구조로 설명한다."
동사 대구. 분석 프로세스 표현.
