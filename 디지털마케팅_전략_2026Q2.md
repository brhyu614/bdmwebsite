# BDM Lab 디지털 마케팅 전략

**목표:** "매출 예측"하면 임보람 교수가 떠오르게 만든다
**기간:** 2026년 Q2~Q3
**핵심 자산:** 검증된 예측 실적 3건 + 이미 잘 만들어진 웹사이트

---

## 현재 진단

### 가진 것 (강력함)

| 프로젝트 | 업종 | 핵심 성과 | 스토리 |
|----------|------|-----------|--------|
| 포토이즘 | 포토부스 프랜차이즈 | 97.7% 적중률, ~5천만원 매출 | "어떤 연예인과 콜라보하면 매출이 얼마 나올까?" |
| BBQ 치킨 | 치킨 프랜차이즈 | R² 0.76, 349개 매장 3년 | "이 동네에 매장을 내면 배달/매장 매출이 각각 얼마?" |
| GS 리테일 | 대형마트 | R² 0.82, JRCS 논문 게재 | "온라인 주문량이 다음 주에 얼마나 될까?" |

추가로 할 수 있는 것: 상권 경쟁력 분석, 폐점 예측, 신규 출점 최적화 등

### 문제 (웹사이트)

| 현재 | 문제 |
|------|------|
| 히어로: "직감 대신 근거. 데이터로 읽는 마케팅." | 뭘 하는 곳인지 5초 안에 파악 불가 |
| 콘텐츠 20편 중 예측 관련 1편 | 인스타 알고리즘, 새벽배송 등 주제가 분산 |
| 네비: 리서치 / 알고리즘 디코드 / 프로젝트 / 연구실 / 교수 / Contact | 학생/연구자용 구조. 잠재 고객용이 아님 |
| CTA: 이메일 한 줄 | "문의하고 싶다"는 흐름이 없음 |
| /projects/photoism 있음 | 좋지만 묻혀 있음. 홈에서 바로 안 보임 |

### 핵심 인사이트

웹사이트를 두 가지 버전으로 나눌 필요 없다.
**교수 연구실 사이트이면서 동시에 "이 사람은 예측을 잘한다"가 자연스럽게 드러나면 된다.**
가격표나 SaaS 느낌 없이, 학술적 권위를 유지하면서.

---

## 전략: "보여주되, 팔지 않는다"

### 원칙

1. **가격, 패키지, "서비스 신청" 같은 표현 절대 안 씀**
2. **프로젝트 실적이 스스로 말하게 함** — 숫자와 스토리로
3. **CTA는 "문의" 하나** — "관심 있으시면 연락 주세요" 수준의 톤
4. **교수/연구실의 학술적 품격 유지** — 이게 진짜 차별화

### 고객이 웹사이트에서 겪는 경험 (이상적 시나리오)

```
1. 네이버에서 "프랜차이즈 매출 예측" 검색
   → BDM Lab 블로그 글 발견 (포토이즘 사례)

2. 글을 읽음 → "97.7%? 진짜?" → 웹사이트 방문

3. 홈페이지 → "아, 한양대 교수 연구실이구나"
   → 프로젝트 3개 보임 (포토이즘, BBQ, GS)
   → "이 교수 진짜 여러 업종에서 해봤네"

4. 포토이즘 케이스 스터디 읽음 → "우리 회사도 되겠는데"

5. 인스타 확인 → "활발하게 활동하고 계시네, 진짜인 듯"

6. 이메일 보냄 → 미팅 → 계약
```

**이 흐름을 만드는 게 전략의 전부다.**

## 실행 계획

### 1단계: 웹사이트 리포지셔닝 (이번 주)

#### 홈페이지 변경

**현재:**
> 직감 대신 근거. 데이터로 읽는 마케팅.
> 디지털 마케팅 매거진.

**변경안:**
> 직감 대신 근거. 데이터로 읽는 마케팅.
> (이건 유지 — 좋은 카피)

아래에 **프로젝트 하이라이트 3개를 홈에 바로 노출:**

```
━━━ Recent Projects ━━━

[포토이즘]     97.7% 적중률 · 462명 연예인 매출 예측
[BBQ 치킨]     349개 매장 · 배달/매장/픽업 채널별 예측
[GS 리테일]    온/오프라인 수요 예측 · JRCS 게재

→ 프로젝트 전체 보기
```

현재 홈은 아티클 그리드만 보여줌.
**프로젝트 하이라이트를 아티클 위에 넣으면, 방문자가 "여기가 뭘 하는 곳인지" 바로 파악.**

#### 네비게이션 변경

**현재:** 리서치 | 알고리즘 디코드 | 프로젝트 | 연구실 | 교수 | Contact

**변경안:** 프로젝트 | 인사이트 | 연구실 | 교수 | 문의

- "프로젝트"를 맨 앞으로 (가장 중요)
- "리서치"와 "알고리즘 디코드"를 "인사이트"로 통합 (외부인에게 시리즈 구분은 의미 없음)
- "Contact"를 "문의"로 (한국어 사이트니까)

#### /projects 페이지 재구성

현재 6개 프로젝트가 나열. **예측 프로젝트 3개를 상단에 크게, 나머지는 하단에.**

상단 (히어로급):
1. 포토이즘 — 이미 상세 페이지 있음
2. BBQ 치킨 — 상세 페이지 신설
3. GS 리테일 — 상세 페이지 신설

하단 (Other Research):
- 새벽배송, 멤버십, 자기잠식 등 기존 것들

#### 문의 페이지 강화

현재 /contact: 이메일, 전화, 주소, 인스타 나열.

추가할 것:
```
"데이터 기반 매출 예측, 상권 분석, 소비자 행동 분석에 대해
궁금한 점이 있으시면 편하게 연락 주세요."

brlim@hanyang.ac.kr
```

이게 전부. 폼도 필요 없고, 가격 안내도 없음. **교수에게 이메일 보내는 느낌.**

### 2단계: 케이스 스터디 페이지 제작 (1~2주)

포토이즘 페이지가 이미 훌륭한 템플릿.
같은 구조로 BBQ와 GS를 만든다.

#### BBQ 치킨 케이스 스터디 (/projects/bbq)

```
히어로: "이 동네에 매장을 내면, 매출이 얼마나 나올까?"
부제: 349개 치킨 프랜차이즈 매장, 3년간의 매출 데이터

핵심 지표:
- R² 0.76 (매장 내 식사)
- 349개 매장 / 3년 데이터
- 배달 · 픽업 · 매장 3채널 각각 예측

섹션 1: 문제 — 프랜차이즈 출점은 감으로 한다
섹션 2: 데이터 — 매출 + 리뷰 10만건 + 경쟁사 + 인구 + 날씨 + 경제지표
섹션 3: 발견 — 인구 밀도가 경쟁사보다 매출에 더 중요
         — 배달 매출은 상권 영향을 덜 받음
         — 같은 브랜드 매장 100m 이내 → 자기잠식 최대
섹션 4: 채널별 예측 정확도 비교
섹션 5: "왜 이런 예측이 나왔는지" SHAP 해석
```

#### GS 리테일 케이스 스터디 (/projects/grocery)

```
히어로: "다음 주 매출이 얼마나 될지, 온라인과 오프라인은 다르다"
부제: 284개 매장, 52주 데이터 · JRCS 게재 확정

핵심 지표:
- R² 0.82 (온라인) / R² 0.66 (오프라인)
- 284개 매장 / 52주
- JRCS (Journal of Retailing and Consumer Services) 게재

섹션 1: 문제 — 온라인 주문과 오프라인 방문, 예측 방법이 같을까?
섹션 2: 발견 — 온라인이 오프라인보다 예측하기 쉽다 (구매 관성)
         — 온라인은 "큰 카테고리"가, 오프라인은 "세부 카테고리"가 중요
섹션 3: 고객 수가 가장 강력한 예측 변수 (SHAP 기준 4~6배 차이)
섹션 4: 실무 함의 — 채널별로 다른 예측 모델을 써야 한다
```

### 3단계: 콘텐츠 — 네이버 블로그 + 웹사이트 아티클 (매주 1편)

**목표:** "프랜차이즈 매출 예측" 검색 시 BDM Lab 콘텐츠가 나오게

#### 콘텐츠 캘린더 (8주)

| 주 | 제목 | 네이버 키워드 | 연결 프로젝트 |
|----|------|--------------|--------------|
| 1 | 프랜차이즈 매출을 97.7% 정확도로 맞춘 방법 | 프랜차이즈 매출 예측 | 포토이즘 |
| 2 | 치킨집 매출, 경쟁사보다 인구 밀도가 더 중요한 이유 | 치킨 프랜차이즈 상권분석 | BBQ |
| 3 | 배달 매출과 매장 매출, 예측 난이도가 다른 이유 | 배달 매출 예측 | BBQ |
| 4 | 온라인 주문이 오프라인보다 예측하기 쉬운 과학적 이유 | 온라인 매출 예측 | GS |
| 5 | 같은 브랜드 매장이 가까우면 매출이 줄어드는가? | 프랜차이즈 자기잠식 | BBQ |
| 6 | AI 매출 예측, 어디까지 가능한가 | AI 매출 예측 | 전체 |
| 7 | 신규 출점 전에 데이터로 확인해야 할 3가지 | 프랜차이즈 출점 데이터 | BBQ+GS |
| 8 | 매출 하락의 신호는 데이터에 먼저 보인다 | 매장 매출 하락 예측 | 전체 |

#### 콘텐츠 작성 원칙

1. **학술적이되 읽기 쉽게** — 현재 아티클 톤 유지 (이미 잘하고 있음)
2. **매 글 끝에 관련 프로젝트 페이지 링크** — "이 분석의 전체 결과는 여기"
3. **숫자를 넣되 과장하지 않기** — "97.7%"는 사실이니까 그대로
4. **네이버 블로그 버전은 제목을 키워드 중심으로** — 웹사이트 버전과 제목만 다르게
5. **절대 "서비스 신청하세요" 같은 CTA 안 넣음** — 글 자체의 퀄리티가 영업

#### 기존 콘텐츠 활용

이미 있는 20개 아티클 중 관련 있는 것:
- `forecasting-restaurant-sales` — BBQ 프로젝트 요약 (이미 있음, 강화)
- `intra-brand-cannibalization` — 상권 분석 관련 (이미 있음)
- `location-data` — 위치 데이터 (이미 있음)

**이것들을 새 프로젝트 페이지에서 "더 읽기"로 연결하면 콘텐츠 간 네트워크가 생김.**

### 4단계: 소셜 미디어 (병행)

#### 인스타그램 @bdm.lab

**역할: "이 사람 진짜인가?" 확인용 + 가벼운 유입**

프로필 변경:
```
현재: (학술 연구실 소개)
변경: AI로 매출을 예측합니다 | 한양대 임보람 교수
      포토이즘 97.7% · BBQ · GS 리테일
      bigdatamarketinglab.com
```

콘텐츠 (주 2~3회):
- **캐러셀 (carousel-generator 이미 있음):** 프로젝트 핵심 발견 요약 (5~7장)
  - "349개 치킨집 3년 데이터로 발견한 것 5가지"
  - "온라인 매출이 예측하기 쉬운 이유"
  - "97.7% 매출 예측, 어떻게 가능한가"
- **릴스 (reels-generator 이미 있음):** 30초 인사이트
- **기존 알고리즘/마케팅 콘텐츠:** 계속 유지 (30% 비율)

**핵심: 예측 프로젝트 콘텐츠 비율을 현재 10% → 70%로**

#### 링크드인 (교수님 개인 프로필)

**역할: B2B 의사결정자 도달 (프랜차이즈 본사 임원급)**

주 1회 포스트:
- 프로젝트 스토리텔링 ("포토이즘과 6개월간 매출 예측 시스템을 만들었습니다")
- 연구 발견 ("배달 매출은 상권보다 XX에 더 영향받습니다")
- JRCS 논문 게재 소식
- 한/영 혼용 가능 (국내 + 글로벌 도달)

#### 네이버 블로그

**역할: 검색 유입의 핵심 채널 (한국 B2B 담당자는 네이버에서 검색)**

- 웹사이트 아티클을 네이버 버전으로 변환 (naver-blog-generator 이미 있음)
- 제목은 검색 키워드 중심으로 재작성
- 매주 1편, 웹사이트 발행과 동시

### 5단계: 중기 — 권위 강화 (2~3개월 후)

#### 콘텐츠 확장

- **프랜차이즈 업계 매체 기고** — 한국프랜차이즈산업협회 뉴스레터 등
- **강연/웨비나** — "데이터로 보는 프랜차이즈 출점 전략" 주제로 무료 웨비나 1회
- **책** — 급하지 않게 병행. 웹사이트 아티클이 쌓이면 그게 챕터가 됨

#### 프로젝트 추가 생산

교수님이 "금방 더 만들 수 있다"고 하셨으므로:
- **상권 경쟁력 분석** — BBQ 데이터에서 상권 부분만 뽑아서 별도 프로젝트
- **매장 폐점 예측** — 기존 데이터에서 하락 매장 패턴 분석
- **리뷰 기반 매출 예측** — BBQ 리뷰 10만건 분석 결과

프로젝트 3개 → 5~6개가 되면, 프로젝트 페이지 자체가 포트폴리오가 됨.

## 채널별 역할 요약

```
[네이버 블로그]  →  검색으로 발견  →  "프랜차이즈 매출 예측 AI"
     ↓
[웹사이트]  →  신뢰 확인  →  프로젝트 케이스 스터디 3개
     ↓
[인스타그램]  →  "진짜인가?" 확인  →  활발한 활동, 숫자 중심 콘텐츠
     ↓
[링크드인]  →  임원급 도달  →  교수 개인 브랜딩
     ↓
[이메일 문의]  →  미팅  →  계약
```

| **7** | 콘텐츠 캘린더 8편 집필 시작 | 매주 반나절 | 검색 키워드 점령 |
| **8** | 링크드인 포스트 시작 | 주 30분 | B2B 도달 |
